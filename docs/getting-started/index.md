# 快速开始提示词

> 复制以下内容给 AI，快速生成项目架构。

---

## 通用项目创建提示词

```markdown
请帮我创建一个新项目，遵循以下架构规范：

### 项目基本信息
- 项目名称：[项目名]
- 项目类型：[全栈应用 / 前端应用 / 后端API / CLI工具]
- 技术栈：[参考技术栈配置文档]
- 主要功能：[简要描述]

### 必须包含的架构要素
1. Monorepo 结构（如适用）
2. 统一的目录结构规范
3. 代码规范配置（ESLint/Prettier）
4. 统一的错误处理和响应格式
5. 认证鉴权框架
6. 数据库迁移机制
7. 测试分层策略
8. 开发文档结构

请按照以下顺序生成：
1. 先创建目录结构
2. 配置核心配置文件
3. 实现基础框架代码
4. 添加示例模块
5. 编写开发文档
```

## 场景化提示词

### 场景 1：创建全栈 SaaS 应用

```markdown
请创建一个 SaaS 多租户管理系统：

技术栈：
- 后端：Node.js + Hapi + Sequelize + MySQL + Redis
- 前端：React + Vite + Ant Design + TypeScript

功能模块：
- 租户管理
- 用户管理（RBAC）
- 计费与订阅

请按照 scaff-ai 模板的目录结构和代码规范实现。
```

### 场景 2：创建前端管理后台

```markdown
请创建一个 React 管理后台：

技术栈：React 18 + Vite + TypeScript + Ant Design

页面：
- 登录页
- 仪表盘
- 用户管理
- 设置页

请包含：
- HTTP 服务封装
- 路由配置
- 权限控制
- 状态管理
```

### 场景 3：添加新功能模块

```markdown
请在现有项目中添加"通知中心"功能，遵循项目现有架构：

1. 数据模型：通知记录表
2. API：发送通知、查询列表、标记已读
3. 前端：通知铃铛组件、通知列表页
4. 后台：异步发送 Worker

请参考现有代码风格和目录结构。
```

### 场景 4：创建 CLI 工具

```markdown
请创建一个 Node.js CLI 工具：

功能：
- 项目初始化脚手架
- 交互式配置
- 模板下载

技术栈：
- commander（命令解析）
- inquirer（交互式问答）
- chalk（彩色输出）
- ora（加载动画）

请按照标准 npm 包结构组织代码。
```

### 场景 5：创建微服务

```markdown
请创建一个微服务项目：

服务列表：
- gateway：API 网关
- user-service：用户服务
- order-service：订单服务

技术栈：
- Node.js + Fastify
- gRPC 通信
- Redis 缓存
- MySQL 数据库

请使用 Monorepo 结构组织。
```

## 迭代优化提示词

### 代码审查

```markdown
请审查当前项目代码，检查以下方面：
1. 是否符合目录结构规范
2. 代码风格是否一致
3. 错误处理是否完善
4. 是否存在安全隐患
5. 测试覆盖是否充分

给出具体的改进建议。
```

### 性能优化

```markdown
请分析当前项目的性能瓶颈：
1. 数据库查询优化
2. 缓存策略
3. 前端加载优化
4. API 响应时间

给出优化方案和实施步骤。
```

### 重构建议

```markdown
请分析当前代码结构，给出重构建议：
1. 模块拆分
2. 代码复用
3. 依赖解耦
4. 架构升级

优先级排序并说明理由。
```
